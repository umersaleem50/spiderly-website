## Configure Google Client ID

### 1. Create an OAuth 2.0 Client ID in Google Cloud

1. Open the [Google Cloud Console — Credentials page](https://console.cloud.google.com/apis/credentials). Create or select a project.
2. Click **Create credentials → OAuth client ID**. If prompted, configure the OAuth consent screen first (User type: **External**, fill in app name, support email, and developer email).
3. Choose **Application type: Web application** and give it a name.
4. Under **Authorized JavaScript origins**, add the origins your admin panel runs on (e.g. `http://localhost:4200` for local dev, `https://admin.example.com` for production).
5. Click **Create** and copy the **Client ID** — a string ending in `.apps.googleusercontent.com`.

### 2. Set the Client ID in your app

The same Client ID goes into two files:

**Backend** — `Backend/YourAppName.WebAPI/appsettings.json`, under `AppSettings.Spiderly.Security`:

```json
{
  "AppSettings": {
    "Spiderly.Security": {
      "GoogleClientId": "1234567890-abc...apps.googleusercontent.com"
    }
  }
}
```

For local development, put the value in `appsettings.Development.local.json` (gitignored). For production, set the environment variable `AppSettings__Spiderly.Security__GoogleClientId` — see [Deployment — Environment Variables](/docs/deployment#environment-variables).

**Frontend** — `Frontend/src/environments/environment.ts`:

```ts
export const environment = {
  // ...
  googleClientId: '1234567890-abc...apps.googleusercontent.com',
};
```
